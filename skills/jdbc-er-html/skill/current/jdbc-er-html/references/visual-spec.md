# ER HTML 视觉规格

本规格继承 `zhimeng-md-html` 默认主题：米白纸感、赤陶橙 accent、高信息密度、克制工程文档风。除非用户明确要求，不要改成深色、hero、黑客感或营销页风格。

## 色板

| 用途 | 值 |
|---|---|
| 纸面 | `#F7F2E5` / `#FBF7EC` / `#EFE8D5` |
| 文字 | `#1F1A14` / `#5A4A38` / `#8B7B62` |
| 分隔线 | `#C9BFA8` / `#E5DDC8` |
| accent 赤陶橙 | `#C15F3C` / `#8B3F25` / `#E8C9B8` |
| MySQL | `#D66F48` |
| PostgreSQL + pgvector | `#496D8F` |
| OSS | `#99733E` |
| ELK | `#675F6F` |
| Redis | `#9D4E44` |
| 多态逻辑 FK / 非规范关联注释 | `rgba(153, 115, 62, 0.58)` |

## 字体

- 标题：中文衬线优先，`Noto Serif SC` / `Source Han Serif SC` / Georgia。
- UI、按钮、表头：系统无衬线，接近 Inter。
- 表名、字段名、类型、代码：等宽字体，接近 JetBrains Mono。

## 布局

- 左侧栏宽度：`320px`。
- 主区域：顶部状态栏 + 大画布。
- 画布世界尺寸：`1900px × 1320px`。
- 普通节点宽度：`240px`。
- 紧凑节点宽度：`220px`。
- 右下详情面板宽度：`420px`。
- 卡片圆角不超过 `8px`。
- 背景使用米白网格，不使用深色背景，不使用彩色渐变背景。

## 左侧顺序

1. 标题和短说明
2. KPI 指标
3. 视图模式
4. 画布
5. 搜索
6. 图例
7. 表清单

## 顶部状态栏

必须显示：

- 库名
- MySQL 版本
- 注释逻辑 FK 数
- 多态逻辑 FK 数：只在大于 0 时显示，不使用红色警示
- 非规范关联注释数：只在大于 0 时显示，并用红色警示
- 物理 FK 异常数：只在大于 0 时显示，并用红色警示
- 短 `schema_hash`

左侧 KPI 不显示库名和 MySQL 版本，避免长文本挤占阅读区；库名和版本只放右上角状态栏。
不要把完整 hash 放在左侧 KPI 大卡里。
完整 hash 必须保留在 `schemaSnapshot.schema_hash_full` 或 `summary.schema_hash_full`。

## Tab 名称

- `业务域分区图`
- `数据落点与协作图`
- `演示紧凑图`

## 关系线

- 注释逻辑 FK：只来自字段注释里的标准格式 `FK -> 目标表名.目标字段名`，用低饱和绿色虚线；默认略淡，选中表后再高亮相关关系。
- 多态逻辑 FK：来自字段注释里的 `多态逻辑FK：条件 -> 目标表.目标字段`，属于合法特殊关系，单独统计，详情中列出；默认不画主关系线，避免误导为普通 FK。
- 非规范关联注释：只作为数据质量提示，不画关系线。
- 物理 FK：不画关系线；如果存在，视为 DDL 异常，校验不得通过。
- 没有注释逻辑 FK 时，不要伪造关系线。

## 外部数据落点

如果当前库里没有 PostgreSQL、OSS、ELK、Redis 对应表，只能在 `数据落点与协作图` 中显示为边界说明，不得伪造成真实表节点。
