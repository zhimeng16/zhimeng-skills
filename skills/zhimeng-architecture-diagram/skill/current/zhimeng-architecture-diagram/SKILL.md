# 芝梦固定版架构图

当用户希望为任意项目生成稳定、高信息密度、可演示的系统架构图 HTML 时使用本 Skill。目标不是生成一次性的通用自动布局图，而是生成可长期复用、风格稳定、信息密度高的固定版架构图。

## 核心原则

稳定效果来自固定的视觉合同：

- 如果当前项目已有认可版 HTML，则先以它作为视觉锚点。
- 如果没有视觉锚点，则按本 Skill 的默认版式生成米白纸感固定版 SVG 架构图。
- 架构事实必须从当前项目资料中抽取，不写死某一个项目的技术栈、范围或优先级。
- 优先使用固定槽位和明确坐标，不使用自由自动布局。

## 资料规则

- 只读取用户指定的资料，以及必要的正式项目文档。
- 如果用户说明某个目录已归档或禁止使用，不得把它作为来源或输出位置。
- 不覆盖已认可的视觉锚点，除非用户明确要求。
- 输出独立 HTML 文件，文件名要和锚点区分开。
- 如果资料之间存在冲突，用紧凑的决策表暴露冲突，不要静默选择。

## 视觉合同

除非用户指定其他风格，否则保留这些特征：

- 主题：米白纸感背景、浅米面板、深色正文、赤陶橙强调色、金色/绿色辅助状态色。
- 推荐 CSS 变量：`--paper:#F7F2E5`、`--soft:#FBF7EC`、`--deep:#EFE8D5`、`--ink:#1F1A14`、`--muted:#6D604E`、`--rule:#C9BFA8`、`--accent:#C15F3C`、`--accent-strong:#B0382F`、`--gold:#B8923A`、`--green:#6B7558`。
- 页面宽度建议：`max-width:1680px` 左右。
- 架构图放在 hero/摘要之后的第一个主模块。
- 使用 inline SVG、固定坐标、宽画布，默认 `viewBox="0 0 1440 1060"`。
- 图上提供 `Fit`、`100%`、`+`、`-` 控件，并支持在图容器内拖拽查看。
- 每个节点包含：英文/系统标识、中文含义、优先级或阶段标签、可选技术/类型标签。
- 连线使用明确的 SVG path。同步调用、异步事件、策略/安全、数据/存储等连线要有视觉区分。
- 文本保持紧凑：一行英文/系统名，一行中文含义，可选一行短说明。发现重叠时调整坐标或节点宽度，不要粗暴压缩字体。
- 图后保留高密度辅助区：关键决策、主链路、边界表、验收门禁、折叠附录或后续专项。

## 默认分层

默认使用横向分层结构。可按项目领域改名，但层次顺序保持一致：

1. 用户/入口层
2. 业务/API 控制层
3. 核心服务/运行时层
4. 数据/存储/集成层
5. 部署/治理/可观测层
6. 后续演进/企业增强层

如果项目不是软件或 SaaS，也要保留类似层次：入口 -> 控制 -> 执行 -> 数据/资源 -> 治理 -> 演进。

## 信息抽取

编辑前先整理一张紧凑节点表：

| 分层 | 英文/系统标识 | 中文含义 | 优先级/阶段 | 类型 | 来源 | 主要连线 |
|---|---|---|---|---|---|---|

再根据节点表生成架构图。

优先级标签由当前项目决定。资料里有 `P0/P1/P2`、`MVP/Next/Later`、`Now/Next/Future` 等规则时，沿用项目规则。不要发明和需求冲突的优先级含义。

## 工作流程

1. 确认是否存在视觉锚点；有锚点则沿用锚点骨架，没有锚点则使用本 Skill 默认视觉合同。
2. 阅读当前需求和架构输入。用户不看 Markdown 时，优先使用 Excel、HTML、表格资料。
3. 抽取节点、边界、数据/存储职责、外部集成点、部署口径、风险门禁和后续演进项。
4. 生成或修订 HTML：复制锚点骨架，只改必要的文本、节点、坐标、连线和辅助表格。
5. 运行校验：
   - `python3 .agents/skills/zhimeng-architecture-diagram/scripts/verify_arch_html.py "<html-path>"`
   - 如果当前项目有硬口径，用 `--required-term` / `--forbidden-term` 增加项目级检查。
   - 可用 Browser/Playwright 打开截图时，必须检查首屏全貌、文字重叠、截断和拖拽/Fit 是否可用。
6. 如果发现重叠、截断、缺少标签、缺少控件、拖拽失效、项目口径漂移，必须修正后再交付。
