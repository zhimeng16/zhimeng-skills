# 复用提示词: 赛博主神殿古典主义

```text
请使用“赛博主神殿古典主义”风格生成一个新的 HTML 页面。

只迁移视觉语言, 不复制任何第三方品牌、logo、原图、字体文件、视频、下载链接或原文案。

视觉语言:
高纯度电蓝背景、白色/米白文字、古典高对比衬线大标题、等宽大写小标签、纪念碑式 hero、铜版画/蚀刻画/古典雕塑感视觉资产、蓝白单色反相、紧凑功能网格、白底蓝字 feature panel、巨大 footer 字标。整体像古典神庙海报被接入赛博终端, 不像普通 SaaS 官网。

页面结构:
1. 顶部仪式化导航: 中央项目名, 两侧少量链接。
2. 首屏 hero: eyebrow + 三行大标题 + 白底蓝字 CTA + 右侧自有或生成的单色古典图像。
3. 中段 showcase: 产品截图、视频帧或明确占位。
4. 3 个平台/能力卡: 蓝底单色图、等宽标签、衬线标题、白底按钮。
5. 白底蓝字功能区: 3 列 x 2 行或移动端单列。
6. footer: 大字标或短宣言, 可少量使用酸黄色强调。

资产规则:
- 如果没有自有图片, 使用 CSS 占位或生成图提示词, 不热链原站资源。
- 生成图方向: monochrome cobalt-blue classical engraving, mythic messenger, temple rays, etched lines, celestial instrument, no logo, no text.
- 不使用来源网站品牌、机构名或任何原站标识。

实现要求:
- 优先单文件 HTML, 内联 CSS, 少量 JS 可选。
- 不依赖远程字体、远程图片或 CDN。
- 支持桌面和移动端。
- 文本不能溢出按钮、卡片或视口。
- 支持 prefers-reduced-motion。

禁区:
- 不要紫蓝渐变、玻璃拟态、圆角 SaaS 卡片、emoji 图标、客户 logo 墙、普通科技感插画。
- 不要把页面做成红杉纸面批注风、米白报告风或后台仪表盘。
- 不要复制参考网站的信息架构和文案。
```

## 验收清单

- [ ] 第一眼是否是电蓝白字、古典神庙、铜版画、实验产品发布页?
- [ ] 是否没有第三方品牌、logo、原图、字体文件和热链资源?
- [ ] hero 是否有纪念碑式三行大标题?
- [ ] 小标签是否是等宽大写并有明显字距?
- [ ] feature panel 是否使用白底蓝字反相?
- [ ] 移动端是否单列且标题不溢出?
